# Security & Responsible Use

- Do not target live systems without authorization.
- Use only approved datasets; respect content boundaries and local laws.
- Scrub PII and secrets from logs and reports before sharing.
- If you discover a vulnerability while using this framework:
  1. Avoid publishing exploit details.
  2. Coordinate disclosure with the affected vendor.
  3. Open a private security issue or contact the maintainer.

## Reporting a Security Issue
Open a **private** security advisory on GitHub or email the maintainer listed in the repo profile. Include minimal reproduction info; do not include keys or personal data.
